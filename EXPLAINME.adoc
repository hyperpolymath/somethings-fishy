This file maps the project's claims to real files, and is honest about which
claims are currently aspirational. The full design is in
`docs/decisions/0001-purpose-and-scope.adoc`.

== What this project claims to be

An agent-autopsy forensic tool. Point it at a repo, a file set, or an estate,
and it reconstructs which agents (and/or humans) have been editing there, what
they did, and what residues they left. Evidence accumulates in VeriSimDB.

It is **not**: a repo linter, an RSR-compliance checker, an AI-generated-code
detector, or an auto-remediation tool.

== Hard invariants (v1)

[cols="1,3", options="header"]
|===
| Invariant                              | Where enforced
| Touch nothing — clone-first analysis   | Engine must refuse targets it has not cloned into the isolation area (pending)
| Writes only via feedback-o-tron        | No direct output sinks compiled into the core (pending)
| Bundled scanners, not reimplemented    | `panic-attacker` invoked as a subprocess (pending)
| Safe to set loose live                 | No destructive code paths; reviewed before every release (pending)
|===

Every "pending" above is a v1 acceptance criterion, not a nice-to-have.

== Central Session Protocol Authority

The session dispatcher (`session/dispatch.sh`) is inherited from the template
and resolves canonical commands against the central standards repo. Nothing in
this project overrides that behaviour.
